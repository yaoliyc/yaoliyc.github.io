Registering with Eureka

当客户向Eureka注册时，它会提供有关自身的元数据 - 例如主机，端口，运行状况指示器URL，主页和其他详细信息。Eureka从属于服务的每个实例接收心跳消息。如果心跳故障超过可配置的时间表，则通常会从注册表中删除该实例。

默认应用程序名称（即服务ID），虚拟主机和非安全端口（取自Environment）分别是${spring.application.name}和${server.port}​。

在类路径上使用spring-cloud-starter-netflix-eureka-client使应用程序成为Eureka“实例”（即，它自己注册）和“客户端”（它可以查询注册表以查找其他服务）。实例行为由eureka.instance.*配置键驱动，但如果您确保应用程序具有spring.application.name的值（这是Eureka服务ID或VIP的默认值），则默认值很好。

更多配置选项查看EurekaInstanceConfigBean和EurekaClientConfigBean。

要禁用Eureka Discovery Client，可以将eureka.client.enabled设置为false。当spring.cloud.discovery.enabled设置为false时，Eureka Discovery Client也将被禁用。

注册安全应用程序

通过设置EurekaInstanceConfig中的两个flags来启用HTTPS：

  • eureka.instance.[nonSecurePortEnabled]=[false]

  • eureka.instance.[securePortEnabled]=[true]

这样做会使Eureka发布实例信息，显示对安全通信的明确偏好。对于以这种方式配置的服务，Spring Cloud DiscoveryClient始终返回以https开头的URI。同样，当以这种方式配置服务时，Eureka（本机）实例信息具有安全的运行状况检查URL。

Eureka健康检查

默认情况下，Eureka使用客户端心跳来确定客户端是否存活。除非另有说明，否则Discovery Client不会根据Spring Boot Actuator传播应用程序的当前运行状况检查状态。因此，在成功注册后，Eureka始终宣布应用程序处于“UP”状态。通过启用Eureka运行状况检查可以更改其行为，从而将应用程序状态传播到Eureka。因此，每个其他应用程序都不会向“UP”以外的状态下的应用程序发送流量。示例：

注册一个服务为什么慢

作为实例还涉及到注册表的定期心跳（通过客户端的serviceUrl），默认持续时间为30秒。在实例，服务器和客户端在其本地缓存中都具有相同的元数据之前，客户端无法发现服务（因此可能需要3次心跳）。可以通过设置eureka.instance.leaseRenewalIntervalInSeconds改变周期。将其设置为小于30的值会加快使客户端连接到其他服务的过程。在生产中，最好坚持使用默认值，因为服务器中的内部计算会对租赁续订期做出假设。

Eureka Clients及时刷新

默认情况下EurekaClient是可刷新的，这意味着可以更改和刷新Eureka客户端属性。当刷新发生时，客户端将从Eureka服务器取消注册，并且可能存在短暂的时间，其中给定服务的所有实例都不可用。可通过设置eureka.client.refresh.enable=false来禁用这一特性。

High Availability, Zones and Regions

Eureka server没有后端存储，所有注册的服务示例必须发送心跳保存注册信息的更新。客户端也在内存中保存Eureka注册信息。

默认情况下，每个Eureka服务端同时也是一个客户端需要一个地址来注册。如果不提供，服务也可以正常运行但日志中会有许多提示无法注册。

独立模式

两个缓存（客户端和服务器）和心跳的组合使得独立的Eureka服务器能够非常适应故障，只要有某种监视器或弹性运行时（例如Cloud Foundry）使其保持活动状态。在独立模式下，您可能更愿意关闭客户端行为，以便它不会继续尝试并且无法访问其对等方。以下示例显示如何关闭客户端行为：

Peer Awareness

Eureka可以通过运行多个实例并要求它们相互注册，可以使Eureka更具弹性和可用性

application.yml (Two Peer Aware Eureka Servers).

上述示例中，我们在两个hosts上运行同样的服务中不同的Spring profiles。可以在单个主机上通过配置/etc/hosts解析这些主机名来测试peer awareness。事实上eureka.instance.hostname不是必须的，默认通过java.net.InetAddress来获取。

可以将多个对Peer添加到系统中，并且只要它们通过至少一个edge彼此连接，它们就会在它们之间同步注册。如果Peer在物理上是分开的（在数据中心内或在多个数据中心之间），那么系统原则上可以存在“split-brain”类型的故障。您可以向系统添加多个对等体，只要它们彼此直接连接，它们就会在它们之间同步注册。

When to Prefer IP Address

在某些时候，Eureka对服务发布IP地址而不是主机名。设置eureka.instance.preferIpAddress为true,应用使用IP地址而不是主机名想Eureka注册。

Securing The Eureka Server

可以通过添加spring-boot-starter-security为Eureka服务提供安全防护。默认情况，当classpath中出现Spring Security时将要求每次向应用程序发出请求时都会发送一个有效的CSRF令牌。Eureka客户端通常不会拥有有效的跨站点请求伪造（CSRF）令牌，您需要为/ eureka / **端点禁用此要求。示例：
