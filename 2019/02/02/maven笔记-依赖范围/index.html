笔记 2019/02/02

依赖范围

​ 依赖范围用来控制依赖于三种classpath(编译classpath、测试classpath、运行classpath)的关系，Maven有一下集中依赖范围：

  • compile：编译依赖范围。默认值
  • test：测试依赖范围。典型Junit
  • provided：已提供依赖范围。典型servlet-api,运行时由tomcat等容器提供，无需重复引入
  • runtime：运行时依赖方位。典型为JDBC
  • system：系统依赖范围。对于maven仓库中没有的类库使用，尽量避免使用。

依赖范围与classpath的关系

依赖范围(Scope) 对于编译classpath有效 对于测试classpath有效 对于运行时classpath有效 例子
compile Y Y Y spring-core
test Y JUnit
provided Y Y servlet-api
runtime Y Y JDBC驱动实现
System Y Y 本地的，Maven仓库之外的类库文件

传递性依赖和依赖范围

compile test provided runtime
compile compile runtime
test test test
provided provided provided provided
runtime runtime runtime

依赖调解

  • 第一原则：路径最近者优先
  • 第二原则：第一声明者优先

    1. Used undeclared dependencies found

      意指项目中使用到，但没有显示声明的依赖，属于潜在的风险，因显式声明任何项目中直接使用的依赖

      项目中未使用，但显式声明的依赖，由于denpendency:analyze只分析编译主代码和测试代码所需依赖，这些信息用于参考，去除无用依赖。
